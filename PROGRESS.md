## Run — 2026-07-17 (first build session after the brief)

Took `cue` from a macOS interview-overlay to **Socrates**, a Windows teaching-and-learning
copilot, end to end. Booted and screenshot-verified on Windows 11 (build 26200, 1920×1200
@ 150%). What changed, by workstream:

### Rebrand — done
- `cue` → `Socrates` across `package.json` (name, appId `com.socrates.overlay`, productName,
  description, keywords), the preload bridge (`window.cue` → `window.socrates`), the OS
  window title, `<title>`, the mode system prompts, and the in-UI copy.
- Settings file `cue-data.json` → `socrates-data.json`. **Migration:** `src/store.js`
  reads the new file, and if absent falls back to the old `cue-data.json`, so an existing
  install keeps its API keys through the rename. The old file is left in place; first save
  writes the new one.
- Env var `CUE_NO_PROTECT` → `SOCRATES_NO_PROTECT`; the old name is still honored.
- Renamed the internal `#cue-status` DOM id → `#status-toast` (CSS + renderer).

### Tutor-shaped modes — done (this is the product thesis)
- `src/prompts.js` rebuilt. Every mode is designed **twice**, and the new `role` setting
  (`learning` | `teaching`, default `learning`) swaps the whole prompt set. `runFeature`
  reads `def[role]`.
- Modes: **explain** (Ctrl+Enter, screen), **hint** (Ctrl+H — a Socratic nudge, never the
  answer), **check** (check understanding), **recap** (study notes / coverage review),
  **ask** (composer). The old bluff-through-a-call modes (`assist` handing over a LeetCode
  solution, `say` writing your sentence, `leetcode`) are gone.
- UI: a **Learn / Teach** toggle in the toolbar (inside `#toolbar`, so click-through
  already allows it). Action-button labels swap per role via `MODE_LABELS` in renderer.js.

### Windows: ship + reach it — mostly done
- `package.json` `build.win` (NSIS target, `build/icon.ico`) + an `nsis` block
  (non-oneClick, desktop + start-menu shortcuts). `dist` script is now `--win nsis`.
- **Tray** added in `main.js` (Show/Hide overlay, Quit) — this is the only handle once
  the overlay hides, since `skipTaskbar` is true and `app.dock.hide()` no-ops. "Show"
  recreates the window if it was destroyed. `window-all-closed` no longer quits, so the
  app stays resident in the tray.
- **Icons** generated by `build/make-icons.js` — a dependency-free rasterizer (Node zlib
  only) that draws the logo's pinwheel-in-ring and emits `icon.png` (256), a multi-size
  `icon.ico`, and `tray.png`. Run `npm run icons` to regenerate.
  - **Gotcha found while packaging:** the tray loads `build/tray.png` at *runtime*, but the
    `build.files` glob only bundled `main.js`/`preload.js`/`src`/`renderer`. Without
    `build/tray.png` in `files`, the packaged tray silently falls back to a plain dot.
    Added `build/tray.png` to `files` (the `.ico` and generator stay build-time only).
- **WDA fallback guard:** `warnIfProtectionFallback()` warns if the Windows build < 19041
  (pre-2004), where content protection degrades to a black box instead of hiding.
- Onboarding replaced: macOS permission steps + `x-apple.systempreferences:` links removed;
  new Windows flow (mic is the only permission, `ms-settings:privacy-microphone` deep link,
  the best-effort honesty step, tray + Ctrl shortcuts).

### Robustness — done
- Transcript array capped at `TRANSCRIPT_CAP = 600` turns (was unbounded).
- `runFeature` busy path now sends a status toast instead of silently dropping the call.

## Decisions taken while running autonomously (reversible; flagged for you)

- **Window title = "Socrates".** The brief flagged this as your product call: the window
  handle is enumerable, so the title is a free detection vector. I chose to title it
  honestly rather than ship a disguise — consistent with the anti-bluff, truth-before-
  polish stance. It's a single constant (`WINDOW_TITLE` in `main.js`); change it if you
  want a lower profile.
- **Model defaults** refreshed (Anthropic → `claude-haiku-4-5` / `claude-sonnet-5`,
  Gemini → `gemini-2.0-flash`). These are user-editable in Settings.
